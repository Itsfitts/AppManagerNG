### Material change set (read the diffs to commit cleanly):

1. **`FilterOption.java` ŌĆö regex predicates were silently neutered.**
   Old behaviour wrapped user-supplied regex values in `Pattern.quote()` before `Pattern.compile()`, so e.g. `".*facebook.*"` only matched the literal 9-character string `.*facebook.*`, not anything containing "facebook". The 2026-05-16 iter-23 work added a regex predicate to `TrackersOption` that would have failed in production. The fix removes `Pattern.quote()`, catches `PatternSyntaxException`, and adds the missing `break` between `TYPE_REGEX` and `TYPE_STR_MULTIPLE` switch cases (a fall-through that overwrote `stringValues`). **Highest-impact uncommitted change.**

2. **`InstallTranscript.java` ŌĆö URI redactor bypass.**
   Old redactor only stripped after the first `/` of the path, so `https://host?token=secret` (no path) returned verbatim including the query. New redactor (a) finds authority end at `/`, `?`, or `#` per RFC 3986 ┬¦3.2, (b) strips any `user[:password]@` userinfo prefix, (c) drops query and fragment. New test cases land alongside in `InstallTranscriptTest.java`.

3. **`OnboardingFragment.java` ŌĆö fragment-detach race.**
   Background-thread root-manager detection called `requireContext()` on the worker pool; if the fragment detached between the post and the background run, an `IllegalStateException` crashed the worker thread silently. Fix captures the `Application` context on the main thread before posting.

4. **`AppOpsOption.java` / `TrackersOption.java`** ŌĆö minor cleanups in the iter-23 filter rows; small diff each.

5. **`FilterOptionTest.java` (new)** ŌĆö Robolectric regression coverage for the regex-quote fix and the switch fall-through fix. 89 lines.
