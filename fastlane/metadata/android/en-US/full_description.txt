<p><b>AppManagerNG</b> is a maintained fork of MuntashirAkon's App Manager. It keeps the original power-user scope, uses the unique package name <code>io.github.sysadmindoc.AppManagerNG</code>, and ships a refreshed NG identity, release pipeline, and distribution-friendly build flavors.</p>

<p>AppManagerNG is built for users who need to inspect, repair, back up, and automate Android apps from one place. It works without privilege where Android allows it, and can use Shizuku, ADB, or root for operations that the platform normally reserves for system tools.</p>

<p><b>Core features</b></p>

<ul>
<li>List installed and backed-up apps across users and profiles</li>
<li>Inspect activities, services, receivers, providers, app ops, permissions, signatures, trackers, libraries, manifests, storage, and usage</li>
<li>Launch activities and services, send guarded broadcasts, and query content providers</li>
<li>Install APK, APKS, APKM, XAPK, split APK, and OBB packages</li>
<li>Back up and restore apps, data, rules, extras, roles, permissions, and encrypted archives</li>
<li>Run batch operations, one-click operations, profiles, scheduled routines, and Tasker-compatible intents</li>
<li>Use the debloater, file manager, code editor, shared-preferences editor, logcat exporter, terminal, and tracker scanner</li>
</ul>

<p><b>Privileged features</b></p>

<ul>
<li>Freeze, unfreeze, force-stop, clear data, clear cache, and control battery optimization</li>
<li>Change app ops, revoke runtime and development permissions, and inspect privileged state</li>
<li>Block components with rule import/export support</li>
<li>Back up and restore app data with root, Shizuku, ADB, and fallback paths where supported</li>
<li>Inspect running apps, processes, network policy, SSAID, and system configuration</li>
</ul>

<p><b>Distribution posture</b></p>

<p>The default <code>floss</code> build disables optional third-party online report features at compile time. AppManagerNG has no ads, no analytics, no telemetry upload, and no bundled tracking SDKs. Broad package visibility is required because this is an on-device package manager; the project documents that permission in <code>docs/distribution/package-visibility.md</code>.</p>

<p>Source, license, reproducible-build notes, changelog, and issue tracker are available from the project repository.</p>
