## Background

Gson 2.14.0 (released 2026-04-23, [S177](../../ROADMAP.md)) ships two changes that matter to AppManagerNG:

1. **Built-in `java.time` adapters** ‚Äî Gson now ships first-party type adapters for `java.time.Instant`, `LocalDate`, `LocalDateTime`, etc. on JDK 8+. The `--add-opens` requirement that previously broke `java.time` serialization on JDK 17 toolchains drops away. AppManagerNG runs on Java 8 source/target with desugaring, but our CI matrix uses JDK 17 ‚Äî this removes a future trap.
2. **Strict duplicate-JSON-key handling** ‚Äî A malformed JSON document with `{"foo": null, "foo": "value"}` now throws `JsonSyntaxException` at parse time instead of silently overwriting the first key. Affects three call sites in NG: profile import, the `android-debloat-list` ingester (`StaticDataset.loadDebloatObjects` + `loadSuggestionObjects`), and the operation-history journal serializer (`OperationHistoryExporter.toJson` / `fromJson`). Strictness is the correct posture for all three ‚Äî silently-deduped duplicate keys mask bugs.

Minor security hygiene: `Serializable` removed from internal `TypeToken` impl classes; closes a deserialization gadget surface that Gson maintainers consider best-practice tightening rather than CVE-grade.
